### Activation (A4)

Replaced the Vercel + Fly + Supabase + Upstash + R2 free-tier runbook with a single-VM Oracle Cloud Always-Free deploy story. New `docs/deployment/oracle-vps.md` walks the operator from Oracle signup through an A1 Ampere VM (4 OCPU / 24 GB RAM / 200 GB block, ARM64 Ubuntu 24.04), hardened-host setup, the unmodified `docker-compose.prod.yml` stack, and TLS via the already-containerised Caddy 2 against Let's Encrypt. Added `scripts/oracle-bootstrap.sh` — an idempotent first-boot installer that creates a non-root admin user, hardens sshd, enables ufw + fail2ban, installs Docker Engine + Compose v2 (ARM64), and stashes the operator's chosen domain + ACME email for use when filling `.env.production`. Deleted `docs/deployment/free-tier.md` and rewrote the README "Deploy it" section accordingly. Cloudflare DNS proxy is documented as the recommended next step, not a prerequisite. Target steady-state cost: **$0/mo, forever**.

### Operator runbook (deploy)

1. **Sign up.** Create an Oracle Cloud account at <https://signup.cloud.oracle.com>, pick a non-saturated home region (Frankfurt/London/Ashburn often reject — try Stockholm, Melbourne, or Vinhedo), then create a `VM.Standard.A1.Flex` instance at the full Always-Free 4-OCPU / 24-GB allowance with Ubuntu 24.04 ARM, public IPv4, and security-list rules for 22/80/443.
2. **Bootstrap.** `ssh ubuntu@<vm-ip>` then `curl -fsSL https://raw.githubusercontent.com/ahmedEid1/E-Learning-Platform/master/scripts/oracle-bootstrap.sh | sudo bash` — answer the three prompts (admin user, domain, Let's-Encrypt email). Log out and back in as the new admin user.
3. **Set domain, then up.** Clone the repo, `cp .env.example .env.production`, fill `APP_DOMAIN` + the secrets called out in runbook step 5, point an A record at the VM's public IP, then `docker compose -f docker-compose.prod.yml --env-file .env.production up -d` + `alembic upgrade head` + `python -m app.cli demo-seed`. Caddy obtains the Let's Encrypt cert automatically on the first HTTPS request.
