# Loops 27 + 28 — Public `/eval` + `/eval/methodology` (interview-ready milestone)

**Date:** 2026-05-27
**Scope:** L27 + L28 ship as one batch. L28 is the **interview-ready
milestone** per the post-redesign v7 plan. Two new public surfaces:

- `/eval` — public read-only eval surface (the headline numbers).
- `/eval/methodology` — long-form companion (how the numbers get made).

The pair are deliberately staged: L27 is the at-a-glance screen, L28
is the closing-argument piece a recruiter clicks into.

## What shipped — L27

`apps/frontend/src/app/eval/page.tsx` + `eval-public-view.tsx`. Three
sections:

1. **Hero** — cartouche + "How the tutor scores. Receipts only." +
   a sealed-run-pending banner (dashed border, mono uppercase). The
   banner is intentional honesty: no fake numbers; the page is
   waiting for an admin-promoted run.
2. **Worked example** — the canonical TS-variance question
   (`ts-variance-canonical`) + expected tool path (`retriever`,
   `code_runner`) + "measurement pending" placeholder. The shape
   matches what `/admin/evals` already records — once a sealed run
   exists, this surface populates without code changes.
3. **Suite trends + adversarial refusal-rate** — both render as
   honest-empty cards today. Adversarial card includes the explicit
   note that per-probe outputs are NOT published (disclosure = a
   roadmap of attacks to try).

Footer routes to the methodology page (next section) + an email-me
contact CTA.

## What shipped — L28 (interview-ready)

`apps/frontend/src/app/eval/methodology/page.tsx` +
`eval-methodology-view.tsx`. Five long-form sections, single column,
generous line-height:

1. **What we measure** — three axes (grounding, accuracy, style)
   defined inline, plus the adversarial refusal-rate scalar formula.
2. **How** — LLM-as-judge with a rubric we ship in-repo; eval
   traffic metered under `feature="eval.judge"` so cost is
   visible; explicit acknowledgement of LLM-as-judge biases.
3. **Adversarial corpus** — the 15-probe set design + the
   string-match heuristic + the "we don't publish per-probe
   outputs" disclosure (with a `ShieldCheck` icon for the
   pull-quote).
4. **Known limits** — judge-on-LLM ~70-85% human agreement, small-N
   noise band on a 0-5 scale, frozen-dataset blindspot.
5. **What I'd do differently at scale** — human-grader pool,
   continuous eval against real-user held-out slice, adversarial
   corpus quarterly rotation.

Footer routes back to `/eval` + an email-me CTA.
